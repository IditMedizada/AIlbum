# AI-lbum App - README

## Overview

**AI-lbum** is an intelligent photo album management system that automatically organizes and groups your photos based on detected faces. It enables users to create albums manually or let the system group photos based on face recognition. Users can filter photos by dates, select the number of photos, and download albums directly to their device.

### Key Features
- **Automatic Album Creation**: The app detects faces in the user's photos and automatically creates albums for each person.
- **Manual Album Creation**: Users can manually create albums by selecting specific faces, dates, and the number of photos.
- **Downloadable Albums**: Created albums can be downloaded and saved locally on the user's device.
  
## Project Structure

The project consists of two parts: the **client-side** (mobile app using Flutter) and the **server-side** (using Node.js and Firebase for storage and face recognition).

### Client-side (Flutter)
The Flutter app allows users to:
1. Upload photos to Firebase.
2. Manually create albums.
3. View and download albums.
4. Take photos using the device camera and upload them directly to Firebase.

### Server-side (Node.js)
The server is responsible for:
1. **Face Detection and Recognition**: It processes the uploaded photos using the `face-api.js` library to detect faces and generate face encodings.
2. **Album Management**: Automatically or manually creates albums based on the detected faces, filters photos by date, and manages album deletion.
3. **Metadata Handling**: Keeps track of processed photos by maintaining metadata for each user and photo in Firebase.

## Prerequisites

### Client-side (Flutter)
- [Flutter SDK](https://flutter.dev/docs/get-started/install)
- Firebase account and project set up with Firestore, Firebase Storage, and Authentication.

### Server-side (Node.js)
- [Node.js](https://nodejs.org/en/) v18 or higher
- Firebase Admin SDK and Firestore enabled.
- Required Node.js modules:
  ```bash
  npm install @tensorflow/tfjs-node face-api.js firebase-admin canvas
  ```

## Installation and Setup

### 1. Clone the Repository

```bash
git clone https://github.com/your-username/AI-lbum.git
cd AI-lbum
```

### 2. Set up Firebase

1. Create a Firebase project and enable Firestore, Firebase Storage, and Firebase Authentication.
2. Download the `serviceAccountKey.json` file from Firebase and place it in the `server` folder.
3. Configure the Firebase SDK by setting up the Firebase credentials:
   
   ```javascript
   // firebaseConfig.js (server-side)
   const admin = require('firebase-admin');
   const serviceAccount = require('./path/to/serviceAccountKey.json');
   
   admin.initializeApp({
     credential: admin.credential.cert(serviceAccount),
     storageBucket: 'your-project-id.appspot.com',
   });
   
   const bucket = admin.storage().bucket();
   module.exports = { bucket };
   ```

### 3. Client-side Setup (Flutter)

1. Open the `client` folder and configure Firebase for Flutter using the `firebase_core`, `firebase_auth`, and `firebase_storage` plugins.
2. Add the necessary Firebase configuration in `lib/firebase_options.dart` (generated by the Firebase CLI).

3. Install dependencies:

   ```bash
   flutter pub get
   ```

### 4. Server-side Setup (Node.js)

1. Install the required dependencies for the server:

   ```bash
   cd server
   npm install
   ```

2. Start the server:

   ```bash
   npm start
   ```

### 5. Run the Flutter App

```bash
flutter run
```

## API Endpoints (Server-Side)

### 1. Create Album
- **Endpoint**: `/api/photos/create-album`
- **Method**: `POST`
- **Description**: Creates a new album for the user based on face detection and filters photos by date range.
- **Request Body**:
  ```json
  {
    "user": "userId",
    "startDate": "2024-01-01",
    "endDate": "2024-02-01",
    "numPhotos": 10,
    "albumName": "Vacation Album"
  }
  ```

### 2. Delete Album
- **Endpoint**: `/api/photos/delete-album`
- **Method**: `DELETE`
- **Description**: Deletes an album and all its associated photos from Firebase.
- **Request Body**:
  ```json
  {
    "albumPath": "userId/user_albums/albumId"
  }
  ```

## How It Works

### 1. Uploading Photos
- Users can upload photos via the mobile app, which stores them in Firebase under their user folder.
- Upon uploading, the server processes these photos, detecting faces and generating face encodings.

### 2. Automatic Album Creation
- The server can create default albums based on the faces detected in the user's photos. For each detected face, a new album is created, grouping all photos with that face.

### 3. Manual Album Creation
- Users can manually select specific faces and date ranges to create custom albums via the app. The server retrieves the photos matching the selected faces and date range.

### 4. Downloading Albums
- Created albums can be downloaded to the user's device from Firebase.
